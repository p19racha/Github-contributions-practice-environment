# 🌟 GitHub Contributions Workshop Repo

Welcome to the **Research and Innovation Team Introduction Project** 🚀

This repository is created for our GitHub hands-on workshop to help first-year students learn how contributions work in a real project.

The project is built using **HTML, CSS, and JavaScript**.

Our goal is simple:

> Learn how to contribute to a GitHub project by making small changes and creating a Pull Request.

---

## 📌 What You Can Contribute

You can contribute by doing simple tasks like:

* Adding your profile / intro card
* Improving the UI
* Fixing alignment issues
* Updating styles
* Adding small JavaScript features
* Fixing typos
* Improving responsiveness

---

## 🚀 Steps to Contribute

Follow these steps carefully.

### 1) Fork the Repository

Click the **Fork** button on the top-right corner of this repository.

This creates your own copy of the project.

---

### 2) Clone Your Fork

Copy your forked repository link and run:

```bash
git clone https://github.com/yourusername/github-contributions.git
```

---

### 3) Move into the Project Folder

```bash
cd github-contributions
```

---

### 4) Create a New Branch

Create a branch using your name.

```bash
git checkout -b your-name
```

Example:

```bash
git checkout -b karthik
```

---

### 5) Make Your Changes

Now open the project and make your assigned changes.

Examples:

* Add your intro card
* Improve button style
* Fix spacing
* Add animations

---

### 6) Save Changes

```bash
git add .
git commit -m "Added my contribution"
```

---

### 7) Push Your Branch

```bash
git push origin your-name
```

Example:

```bash
git push origin karthik
```

---

### 8) Create Pull Request

Go to GitHub and click **Compare & Pull Request**

Fill the PR template and submit it.

---

## 📝 Using Templates

Before working, check the **Issues** section.

Pick an issue and work on it.

When creating a Pull Request, please fill the **PR template** properly.

Templates are already added inside the `.github` folder.

---

## 💡 Contribution Rules

* Keep changes simple and clean
* Do not edit others’ work
* Use your own branch
* Raise a Pull Request after completing work
* Mention the issue number in PR if possible

---

## 🎯 Goal of This Workshop

By contributing here, you will learn:

* Forking
* Cloning
* Branching
* Commits
* Pull Requests
* Collaboration workflow

This is how real open-source contributions happen 🚀

Happy Contributing ✨
